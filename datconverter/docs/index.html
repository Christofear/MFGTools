User Guide and Technical Reference

This utility provides a high-speed bridge between database files (.dat) and Microsoft Excel (.xlsx). It is specifically designed to facilitate bulk library management, allowing users to leverage spreadsheet features to build complex tool databases efficiently.

Supported Database Files

This tool is optimized for database files commonly found in the \MACH\resource\library directory, including:

  • cut_methods.dat
  • feeds_speeds.dat
  • holder_database.dat
  • segmented_tool_database.dat
  • shank_database.dat
  • tool_database.dat

Backup Recommendations

Maintain backup copies of your original .dat files before performing bulk updates.

Bulk Library Management with Excel

The primary purpose of this tool is to replace manual data entry with efficient bulk processing.

  1. Initial Export: Generate the template structure from an existing file.
  2. Integration: Copy data directly into corresponding Excel columns.
  3. Mass Editing: Use Excel's features to update hundreds of values simultaneously.
  4. Validation: The tool checks for duplicate Library References (LIBRF) or values exceeding system limits.

Common Errors and Troubleshooting

Error Message Cause Resolution
Duplicate Library Reference Multiple entries share the same LIBRF Ensure every tool has a unique ID unless using rtype 2 exceptions
Missing separator '|' A data row count does not match the FORMAT definition Check that all columns are properly separated

DATA before FORMAT Structural error in source file Ensure the FORMAT line appears immediately after #CLASS

Database Structure

Successful conversion requires strict adherence to the ASCII schema. The header definitions determine how the application allocates memory and validates individual fields.

# Structure Example
#CLASS MILLING_DRILLING
FORMAT LIBRF | DIA | FLEN | RAD | MATREF | CTH | HND
DATA | "EM_10mm" | 10.0 | 45.0 | 0.0 | "CARBIDE" | 1 | 0
#END_DATA
ElementDescription
#CLASSDefines the tool category (e.g., MILLING_DRILLING)
FORMATDefines column headers and sequence
DATA |Prefixes individual records; pipe-separated
#END_DATASignals the conclusion of a class section
# CommentsMetadata, system notes, or headers. Preserves all lines starting with # during conversion.

Configuration Editor

The Configuration Editor allows you to manage tool class requirements, map database headers to Excel display names, and build custom logic to validate data integrity before conversion.

1. Column Mappings

This section is on **Tab 1**. It lists all the column headers commonly found in manufacturing .dat library files.

  • Customization: You can add your own column headers and specify the required data type (Integer or Double).
  • Validation: The application will warn you if data entered in Excel does not follow the specified data type.
  • Descriptions: You can set a description for each field. This will be used to name the column in a more readable way in the Excel export.

2. Class Definitions & Custom Rules

This section is on **Tab 2**. The left side lists common tool classes. Here you can select a class and edit exactly which columns apply to it.

  • Existence Check: By adding a column to a class but leaving the rule blank, the app will check that the column exists in the data and that's it.
  • Custom Rules: By adding a formula to that column, you can set specific rules for the data to ensure integrity.
  • Scenario: For ENDMILL_NON_INDEXABLE, ensure Flute Length (FL) never exceeds Length (LEN) by entering a rule like .
  • Apply: Rules are active immediately. Failure alerts appear in orange in the Warning Log.

Combining Logic (| and &)

  • AND (&): Both true. >0 & <1
  • OR (|): Either true. =5 | =10

3. Why use this?

Prevent "bad data" from ever being converted. If a decimal is entered into an integer field or a flute length exceeds tool length, the application flags the exact line needing correction.

CLI Operational Guide

The Command Line Interface is available for users who wish to automate conversion tasks via batch files or scripts.

# Usage

DatConverter.exe [input_path] [options]

Options

FlagDescription
-o, --output <path>Specify the output file or directory
-y, --overwriteAutomatically overwrite existing files
-q, --quietMinimize console output for cleaner logs
--log <path>Save session output to a text file
--dat-only / --xlsx-onlyRestrict processing to a specific file type

Exit Codes

  • 0 Successful processing and validation
  • 1 Failed. Check the log for validation errors or missing files

Activation Guide

To activate NX Converter, follow these steps:

1 Copy the License Block

Copy the entire text block provided to you, including the header and footer lines.

--- BEGIN DAT CONVERTER LICENSE ---
...
--- END DAT CONVERTER LICENSE ---

2 Paste and Apply

  • Open License window in the application.
  • Paste the block into the text area.
  • Click Activate.

3 Confirm Status

The window will display your registration email and expiration date once the key is recognized. Activation is complete and works entirely offline.

Troubleshooting

  • Invalid Block:
    Ensure you included the BEGIN and END lines when copying the text.
  • Expired:
    Check if your system date matches the current date.
